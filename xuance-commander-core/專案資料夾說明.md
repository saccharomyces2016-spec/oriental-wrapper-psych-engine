# 專案資料夾說明（給人類看的）

## 📖 這份文件的目的

這份文件是給**人類**看的，特別是：
- 沒有程式背景的新手
- 想要了解這個專案在做什麼的人
- 需要快速找到某個文件的人

**簡單來說**：這份文件會告訴你「哪個資料夾裡面放什麼東西、有什麼功用」。

---

## 🗂️ 主要資料夾說明

### 📁 `docs/` - 所有文件的家

**這是什麼**：所有專案相關的文件都放在這裡。

**裡面有什麼**：
- **`docs/governance/`** - 專案規則和制度
  - 就像公司的規章制度，告訴大家「什麼可以做、什麼不能做」
  - 例如：文件要寫在哪裡、任務要怎麼執行、AI 要怎麼協作
- **`docs/domain/`** - 專案的核心知識
  - 就像專案的「知識庫」，存放所有關於產品設計、功能規格的內容
  - 例如：題目設計、敘事文案、安全規則
- **`docs/gem/`** - AI 顧問的工作記錄
  - 就像「顧問師的工作檔案夾」，存放 AI 顧問師的任務和產出
  - 例如：任務包、顧問師的建議稿、執行記錄
- **`docs/ops/`** - 專案運營記錄
  - 就像「專案日記」，記錄任務進度、健康檢查報告
  - 例如：任務記錄、健康檢查結果
- **`docs/adr/`** - 重要決策記錄
  - 就像「決策會議記錄」，記錄為什麼做某個決定
  - 例如：為什麼選擇某個設計方案、為什麼採用某個技術

**超連結**：[docs/](docs/)

---

### 📁 `domain/` - 產品的核心資料

**這是什麼**：存放產品實際運作時需要的資料。

**裡面有什麼**：
- **`domain/compiled/`** - 編譯好的產品資料
  - 就像「成品倉庫」，存放已經處理好、可以直接使用的資料
  - 例如：編譯好的題目、敘事、建議
- **`domain/facets/`** - 不同主題的資料
  - 就像「不同類型的產品」，每個資料夾代表一個主題
  - 例如：收入壓力、關係失衡
- **`domain/manifests/`** - 資料清單
  - 就像「目錄」，告訴系統「有哪些資料、在哪裡」
  - 例如：資料清單、版本資訊

**超連結**：[domain/](domain/)

---

### 📁 `memory/` - 專案的記憶

**這是什麼**：存放專案的「記憶」，記錄專案現在的狀態、做過什麼、接下來要做什麼。

**裡面有什麼**：
- **`memory/briefs/`** - 專案摘要
  - 就像「專案快報」，快速了解專案現在的狀態
  - 例如：當前任務摘要、專案狀態
- **`memory/changes/`** - 變更記錄
  - 就像「更新日誌」，記錄專案做了哪些改變
  - 例如：CHANGELOG.md
- **`memory/index/`** - 索引文件
  - 就像「目錄」，告訴你「重要文件在哪裡」
  - 例如：INDEX.md、快速入口

**超連結**：[memory/](memory/)

### 📁 `specs/` - 規格文件（按特性分類）⭐ **新建**

**這是什麼**：所有規格文件按特性分類存放，取代原本散落的 P0 任務資料夾。

**裡面有什麼**：
- **`specs/ui/`** - UI 相關規格
  - `architecture/` - UI 架構設計
  - `workflow/` - UI 流程設計（4 階段分流系統）
  - `worldview/` - 世界觀設計
  - `interaction/` - 互動設計
- **`specs/engine/`** - 引擎相關規格
  - `core/` - 底層引擎（科學計算）
  - `metaphysical/` - 表層引擎（玄學運算）
- **`specs/domain/`** - Domain 資料規格
- **`specs/integration/`** - 整合規格

**超連結**：[specs/](specs/)

**這是什麼**：所有已完成的任務資料夾、備份檔案、歷史記錄都歸檔到此。

**裡面有什麼**：

  - `engine/` - 引擎相關任務（P0-11, P0-12, P0-5.6）

  - `domain/` - Domain 相關任務（P0-10）
  - `results_presentation/` - 結果呈現相關任務（P0-13）

  - `user_validation/` - 使用者驗證相關任務（P1-1）
  - `deployment/` - 部署相關任務（P5-1）
  - 每個類別包含：
    - `TASK_REPORTS.md` - 整合所有任務報告
    - `TASK_INDEX.md` - 快速查找索引
    - `tasks/` - 原始任務資料夾
- **`archive/p0_tasks/`** - Phase 0 任務資料夾（已歸檔，保留作為備份）
- **`archive/p1_tasks/`** - Phase 1 任務資料夾（已歸檔，保留作為備份）
- **`archive/p5_tasks/`** - Phase 5 任務資料夾（已歸檔，保留作為備份）
- **`archive/backups/`** - 備份檔案
  - `.zip` 備份檔案
- **`archive/governance_duplicates/`** - 重複的治理規則（已歸檔）

**注意**：
- 基準檔案已移動到 `specs/`
- 任務資料已按類別重新組織到 `archive/tasks_by_category/`
- 原始 Phase 結構保留作為備份

**超連結**：[archive/](archive/)

## 🔍 快速查找指南

### 🚀 AI 第一次接觸此專案？
→ **看 `docs/ONBOARDING.md`（AI 上手指南）**
  - 告訴 AI 應該先讀什麼文件
  - 按優先級排序（必須讀 vs 建議讀 vs 需要時查閱）
  - 包含快速開始工作指南

### 我想了解專案的終極目標
→ 看 `charter/CHARTER.md`（SSOT，取代舊的 `FULL/NORTH_STAR.md`）

### 我想了解專案現在在做什麼
→ 看 `docs/ops/TASK_STATUS.md`（任務狀態總覽，統一入口）
→ 或看 `roadmap/ROADMAP.md`（主線進度）

### 我想了解專案的規則和制度
→ 看 `docs/governance/ESSENTIAL_RULES.md`（核心制度，必須遵守的規則）
→ 或看 `docs/governance/GOVERNANCE_INDEX.md`（完整治理索引）

### 我想了解文件應該放在哪裡
→ 看 `docs/governance/CURSOR_FILE_PLACEMENT_RULE.md`（文件放置規範）

### 我想了解專案的核心知識
→ 看 `docs/domain/` 資料夾

### 我想了解某個階段的任務
→ 看 `archive/p0_tasks/P0-X/` 資料夾（例如：`archive/p0_tasks/P0-5/`）
→ 或看 `archive/p1_tasks/`, `archive/p5_tasks/` 等

### 我想了解專案現在的狀態
→ 看 `docs/ops/TASK_STATUS.md`（任務狀態總覽，統一入口）
→ 或看 `memory/briefs/CURRENT.md`（當前狀態摘要）
