2. **進度對齊**：
   - Phase 0：MVP 核心功能的最後一個階段

   - 結案報告已定案，執行報告已建立，所有規格明確

3. **依賴關係對齊**：
   - P1-1（使用者驗證）依賴 P0-5 工程實作完成
   - P5-1（生產環境部署）依賴 P0-5 工程實作完成
   - 完成 P0-5 後可立即啟動 Phase 1 和 Phase 5

**目標**：
- 根據 `P0-5/P0-5_UMIP_CLOSURE_REPORT.md` v1.0 和 `P0-5/P0-5_IMPLEMENTATION_EXECUTION_REPORT.md` v3.0 FINAL 實作 UI
- 滿足 6 項 MVP Gate 驗收標準
- 實現可跨 Facet 重用、可被工程驗收的 UI 整合方案

**關鍵交付物**（8 項）：
1. `schemas/umip_schema_v1.json`（含 `theme_config`、`i18n` 欄位與必要 fallback 約束）
2. `ui/` 目錄（新建標準化工程目錄）
3. Stage 2「Radial Compass」元件（放射狀羅盤/星盤介面，RWD + Touch 支援）
4. `i18nManager`（CN/EN 即時切換引擎，no reload）
5. `ThemeEngine`（`theme_config` 視覺解耦與容錯機制）
6. `RiskOverride`（高風險 L4 強制覆蓋模組）
7. `ResultLayers`（L1→L4 分層呈現，L3 主動揭示）
8. `P0-5/P0-5_IMPLEMENTATION_ACCEPTANCE_REPORT.md`（驗收報告）

**執行順序**（5 個階段）：
1. Foundation：建立 `ui/` 骨架 + `umip_schema_v1.json`
2. Core Engine：`FacetLoader` + `ThemeEngine`（含 Fail-soft）
3. Stage 2：`RadialCompass`（先 mobile 可用，再 desktop 完整）
4. Stage 4：L1–L4 分層呈現 + `RiskOverride`
5. Integration：i18n 全流程整合 + Gate 逐條驗收

**預期時間**：根據工程實作複雜度，預估 2–4 週

**驗收標準**：必須滿足執行報告第 6 章的 6 項 MVP Gate（Audit-Ready Checklist）

## 🔄 回滾機制（Rollback Strategy）

### 若需回滾至先前狀態

**情境 1：技術方案不適用**
- 可回滾：前端框架選擇、項目結構決策
- 回滾方式：更新 ADR 記錄，說明回滾原因與新方案
- 影響範圍：僅影響技術實現，不影響 UMIP 規格

**情境 2：執行順序需要調整**
- 可回滾：執行順序（5 個階段）
- 回滾方式：更新執行報告 Addendum，說明調整原因
- 影響範圍：僅影響開發流程，不影響最終交付物

**情境 3：驗收標準需要放寬**
- 可回滾：6 項 MVP Gate 的具體驗收要求（需謹慎）
- 回滾方式：需先更新結案報告並獲得批准（見執行報告 9.1 節）
- 影響範圍：影響驗收標準，需與專案治理對齊

**情境 4：終極目標需要調整**
- 可回滾：終極目標（需使用者批准）
- 回滾方式：更新 `charter/CHARTER.md`（需使用者批准）
- 影響範圍：影響整體專案方向，需最高層級決策
