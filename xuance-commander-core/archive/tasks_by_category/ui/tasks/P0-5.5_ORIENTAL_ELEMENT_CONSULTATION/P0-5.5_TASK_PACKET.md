## 一、任務概述

### 1.1 任務目標

為我們的產品選定最適合的東方玄學/算命元素作為用戶界面（UI）的表層呈現系統。

**候選元素包括但不限於**：
- 五行（金、木、水、火、土）
- 八卦（乾、坤、震、巽、坎、離、艮、兌）
- 紫微斗數
- 八字
- 其他東方玄學系統

### 1.2 任務重要性

這個選擇將**深刻影響**：
1. **現代科學結合度**：後端科學數據與前端玄學呈現的映射關係
2. **市場定位與競爭力**：國際市場與東方市場的接受度和競爭機會
3. **後續開發方向**：UI 設計、題目設計、用戶體驗的整體方向

### 1.3 評估維度

需要從以下**三個核心維度**進行評估：

1. **市場潛力**（Market Potential）：哪一種最有市場潛力？
2. **市場缺乏度**（Market Gap）：哪一種市場最缺乏高品質產品？
3. **科學結合度**（Scientific Integration）：哪一種最能夠與現代科學進行結合？

## 二、核心交付物

### 2.1 必交付物

1. **顧問諮詢報告**（Consultant Report）
   - 市場分析（國際市場、東方市場）
   - 科學結合度分析
   - 技術實現分析
   - 綜合評估與建議

2. **市場分析報告**（Market Analysis Report）
   - 國際市場分析（英文語境）
   - 東方市場分析（華語、日韓市場）
   - 現有產品競爭分析
   - 市場空白和機會分析

3. **元素選擇決策文件**（Element Selection Decision Document）
   - 最終選定的元素系統
   - 選擇理由（對齊三個評估維度）
   - 實施計劃（如何調整 Stage 1-4 的設計）
   - 風險評估和應對措施

### 2.2 預期時間

- **總預計時間**：1-2 週
- **諮詢輪次**：建議至少 4 輪（背景理解 → 深度分析 → 綜合評估 → 決策確認）

## 三、關鍵背景資料

### 3.1 項目終極目標

**三個核心要求**（必須同時滿足）：
1. **真的能賺錢**（Profitability）
2. **真的可以上國際**（Internationalization）
3. **真的可以幫到人**（User Value）

**核心策略**：「後端科學極大化，前端玄學極致化」

### 3.2 現有項目情況

**當前階段**：Phase 0 MVP 已完成核心功能

**4 階段全像漏斗系統**：
- **Stage 1**：The Wheel（八卦定方位 / 分流選擇）- 已設計，未實作
- **Stage 2**：Radial Compass（六親定物象 / 符碼選擇）- 已實作 UI
- **Stage 3**：Projection（萬象定歸因 / 投射問題）- 已實作 UI
- **Stage 4**：Results（命盤綜合與斷語 / 結果呈現）- 已實作 UI

**數據結構**：通用玄學介面協議（UMIP v1.0）

### 3.3 未來方向

**可移植性要求**：選定的元素系統必須能夠跨 Facet 重用

**技術需求**：
- 支持 CN/EN 雙語，即時切換
- 支持多種不同主題的構面（facet）
- 支持結構化數據映射

## 四、關鍵決策問題

### 4.1 關於市場定位

- 國際市場 vs 東方市場：應該優先考慮哪個市場？
- 市場成熟度：哪種元素系統的市場最成熟？最缺乏高品質產品？
- 競爭優勢：在哪種元素系統上，我們最有可能建立競爭優勢？

### 4.2 關於技術實現

- 結構化映射：哪種元素系統最能與我們的結構化數據無縫結合？
- 可擴展性：哪種元素系統最能支持多種不同主題的構面？
- 用戶體驗：哪種元素系統最能提供良好的用戶體驗？

### 4.3 關於科學結合

- 邏輯一致性：哪種元素系統最能與現代心理學和相關科學系統保持邏輯一致性？
- 敘事框架：哪種元素系統最能支持豐富的敘事框架？

### 4.4 關於具體實施

- Stage 1-4 的設計調整計劃
- 題目設計調整計劃
- 結果呈現調整計劃
