# System Capabilities（系統能力聲明）

本文件用來宣告「系統已納入設計視野、但未必於當前階段實作」的能力。
此處內容屬於能力邊界與方向聲明，不構成實作承諾。

---

## Capability A：Analysis Orchestrator（AI 聚合分析層）

定義：
系統在完成題目分析後，必須能將「結構化診斷結果」送至外部 AI（依 API KEY 指定），
由該 AI 進行跨層整合、敘事放大與多視角呈現。

約束：
- 題目系統負責診斷，不負責最終敘事
- 外部 AI 僅能做整合與呈現，不得回寫診斷結果
- 顯示內容可調整揭露比例，但不得影響診斷準確性

---

## Capability B：Exploration / Unlock Mechanism（探索／抽取式呈現）

定義：
系統可加入解鎖、抽取或探索機制，以提高使用者持續互動動機。

約束：
- 此能力僅影響呈現順序與節奏
- 不得干擾或改寫診斷邏輯
- 成癮感屬 UX 層，非核心引擎
