# Cursor Limitation Review Rule（人工/策略層檢查）

## 為什麼需要
Cursor 在寫檔當下可處理定位/重複/語法/即時驗證，但**無法自動判斷**下列三類「策略/語義」風險。

## 三類必回檢項（SSOT）
1) **語義層衝突**
   - 不同文件概念重疊、定義不一致（非語法錯誤）
2) **跨文件策略衝突**
   - TEXT_ONLY / TASK_LIFECYCLE / GOVERNANCE 規則取捨與優先序
3) **歷史意圖偏離**
   - 新規則是否違反既有治理決策（ADR/INCIDENT）

## 何時檢查（強制）
- 每次「治理階段」結束
- 每次「規則新增 ≥ 2 份」
- 每次「準備推進主線任務」前

## 怎麼檢查（Cursor 任務）
- 以 **Cursor 掃描** 列出上述三類風險清單（僅列舉，不改檔）
- 輸出到 `VERIFICATION_PACK`
- MASTER_MIN 僅保留結論摘要（≤ 8 行）

## 驗收
- 三類皆有結論（OK / 需決策 / 已修）
- 若需決策，必在 CURRENT 註記下一步
