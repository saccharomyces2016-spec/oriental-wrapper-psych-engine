# Execution Assignment Rule（任務指派決策規範｜硬規則）

## 目的
避免因指派錯誤導致的返工、延誤與品質波動。

本規範定義：**在多 AI 協作環境中，任務應如何被指派給正確的執行者。**

---

## 一、決策總原則（只問一個問題）

> **「這件事是否要求一次就對、可驗收、直接落盤？」**

- 是 → **Codex**
- 否 → **Cursor**

---

## 二、Codex 指派條件（任一成立即指派）

適用於：
- 腳本修復 / 生成
- 治理文件新增或修訂
- 同步、封板、checkpoint 類任務
- 有明確驗收條件（grep / test / status）
- 目標是「一次完成、不可碎裂」

特性：
- 指令完整
- 規格嚴格
- 禁止即興修改

---

## 三、Cursor 指派條件（任一成立即指派）

適用於：
- 探索型任務
- 題目 wording / 敘事草稿
- UI / UX 試排
- 需要人類即時判斷方向
- 可接受多輪修正

備註：
- Cursor 爆量僅影響速度，不影響推理能力
- 允許簡短指令與高互動

---

## 四、指揮官義務（Xuance Commander）

- 必須主動判斷任務性質
- 必須明確告知「此指令應貼給誰」
- 若發現更優分工，需主動提出替代方案
- 不得因節省流量而犧牲結構品質

---

## 五、禁止事項

- ❌ 不得將「一次落盤型任務」交給 Cursor
- ❌ 不得將「探索型任務」強行交給 Codex
- ❌ 不得混用同一任務於兩者
