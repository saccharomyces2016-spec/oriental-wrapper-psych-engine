# Cursor Audit → Governance Upgrade Rule（強制制度升級規則）

## 觸發條件
只要符合任一條件，即視為「治理事件」：
- Cursor 一次性盤點出 **多項同類型錯誤**
- Cursor 報告指出 **結構性／重複性／可預期再次發生** 的問題
- Cursor 需跨多目錄、跨多檔案才能解釋清楚問題來源

## 強制行為（不得跳過）
1. **不得直接修 code**
2. 必須先回答三件事，並寫入文本：
   - 為什麼這類錯誤會反覆出現
   - 現行制度哪一條「不存在或不夠硬」
   - 未來要怎麼讓同類錯誤「根本不會再出現」

## 必產出（缺一不可）
- 新治理規則文件（docs/governance/*）
- 或既有規則的補強修訂
- 對應的檢查方式（audit script / checklist / guard）

## 分工規範
- Cursor：負責盤點、分類、歸納錯誤模式
- 指揮官：負責判斷是否上升為制度
- Codex：只負責在「制度完成後」逐條修正

## 禁止行為
- 禁止「只修這一次」
- 禁止「靠記憶避免下次犯」
- 禁止未留下制度紀錄的快速修補

## 完成條件
- 每一類錯誤都有：
  - 明確規則
  - Canon 路徑
  - 可重跑的驗證方式
- Cursor 重新掃描時，不再回報同類問題
