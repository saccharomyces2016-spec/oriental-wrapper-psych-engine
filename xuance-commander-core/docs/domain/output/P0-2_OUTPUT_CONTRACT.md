Define the system-level output contract that specifies:

## P0-2 Output Contract — Stage Conclusion (DRAFT)

### 1. 核心結論（已確認）
- 本階段目標 **不是** 提供人生判斷或具體行動建議  
- 本階段目標 **僅限於**：  
  **定義引擎「可以說什麼」與「絕對不能說什麼」**

### 2. 引擎的角色定位（已鎖定）
- 引擎只負責 **判斷狀態（State Assessment）**
- 引擎不負責 **判斷人生（Life Judgment）**

### 3. 明確禁止輸出（Hard No）
以下內容在任何輸出層級 **一律禁止**：
- ❌ 客觀財務事實（是否有錢、欠債、收入）
- ❌ 心理 / 醫療診斷（憂鬱、焦慮、創傷等）
- ❌ 將問題歸因於特定人物（婆媳、老闆、配偶）
- ❌ 成敗或命運預測（一定成功 / 一定失敗）
- ❌ 具體人生指令（離職、投資、離婚等）

### 4. 允許的輸出層級（L1–L4）
- L1 狀態映照（像什麼）
- L2 動態趨勢（目前偏守 / 偏動）
- L3 隱患提示（但不點名、不定罪）
- L4 抽象調理方向（宜守 / 宜疏 / 宜緩）

L5（具體生活指令）在 P0-2 **全面禁止**

### 5. 關鍵判斷
- 本階段「限制多於創造」是**正確設計**
- 任何「出口設計」不得破壞上述禁止項

Status: DRAFT / NON-BINDING  
Revision Strategy: 後續僅能新增「替代出口規則」，不得放寬禁止項

# P0-2 Output Contract — Stage Final Summary
Status: **DRAFT / NON-BINDING**
Scope: Engine Output Boundary & Safe Expression Space

## 1. 本階段真正完成了什麼（不是產出文案）
本階段**不是**在設計使用者會看到的內容，
而是在定義「系統在任何情況下**允許說什麼、不能說什麼、必須改說什麼**」。

核心結論：
- 引擎只負責 **判斷狀態（State Assessment）**
- 引擎不負責 **判斷人生（Life Judgment）**

---

## 2. 已鎖定的硬邊界（不可越線）
以下內容在 P0-2 階段被明確禁止輸出，且此禁止是**結構性安全需求**：

- ❌ 客觀財務事實（是否有錢、是否負債）
- ❌ 心理／醫療診斷（憂鬱、焦慮、病理名詞）
- ❌ 將問題歸因到特定人物（家人、伴侶、老闆）
- ❌ 成敗、命運、確定性預言
- ❌ 具體人生或財務指令（離職、投資、分手）

> 備註：禁止 ≠ 刪除輸出，而是 **必須轉譯**

---

## 3. 關鍵補完（R4 的核心貢獻）
本階段最重要的新增價值是：
**「禁止項不是終點，而是轉譯的起點」**

當語意觸及禁止區時，系統必須轉向以下「允許的抽象出口類型」，而非沉默或刪減。

### 六大允許輸出類型（Safe Expression Space）
1. **能量狀態描述**（流動 / 凝滯 / 枯竭 / 亂流）
2. **節奏與循環**（快慢 / 冬藏 / 蓄勢 / 停泊）
3. **場域與摩擦**（阻力 / 風向 / 環境干擾）
4. **結構承載評估**（撐得住 / 過載 / 吃力）
5. **姿態建議**（宜守 / 宜緩 / 宜疏 / 暫停）
6. **覺察提醒**（徵兆已現 / 需提高警覺）

> 這些不是建議動作，而是**描述態勢**

---

## 4. 輸出層級共識（L1–L4）
P0-2 階段僅允許以下結論層級：

- **L1 狀態映照**：現在像什麼
- **L2 動態趨勢**：目前偏守還是偏動
- **L3 隱患提示**：若持續，可能吃力
- **L4 抽象調理方向**：宜守 / 宜疏 / 宜緩

**L5（具體生活指令）在本階段全面禁止**

---

## 5. 本階段刻意「不做」的事
以下內容被確認為 **未進入 P0-2 Scope**，將留待後續階段：

- 玄學語言的銳利度與文學性
- 高命中率的敘事模板優化
- 特定情境（家庭 / 婆媳 / 創業）細分
- UI 呈現與視覺風格

> 原因：邊界未穩前，不進行風格放大
