# 【專案規格書】東方玄學介面 × 心理科學核心｜檢測系統開發計劃（Final Release）

**文件版本：** v4.0 (Master Compilation)  
**核心概念：** 以「易經/八卦」為使用者介面（UI），以「投射測驗/資源管理」為底層運算（Engine）。  
**開發目標：** 讓使用者在「抽籤/算命」的儀式感中，完成精準的心理狀態評估與問題分析，並提供具備安全邊界的策略建議。

---

## 1. 核心邏輯架構

本系統不依賴隨機運氣，而是透過三個同心圓的「過濾器」，解析使用者的決策模型。系統運作遵循以下公式：

* **第一層（入口）：鎖定戰場**（定義問題範疇）。
* **第二層（內圈）：駕駛員掃描**（本質態：心理效能與認知）。
* **第三層（中圈）：路況掃描**（環境態：客觀阻力與急迫性）。
* **第四層（外圈）：行為掃描**（應對態：實際執行的策略）。
* **最終輸出：定錨決策**（Gap Analysis：計算「想做、能做、在做」之間的落差）。

---

## 2. 第一階段：鎖定戰場（Context Layer）

* **介面形式：** 八卦盤（8個圓形按鈕，中心顯示書法單字）。
* **功能目的：** 建立投射邊界，防止歸因錯誤。
* **商業價值：** 分類明確，利於建立「單項付費」或「全盤解鎖」機制。

| 符碼 | 戰場分類 | 涵蓋範疇 |
| --- | --- | --- |
| **【財】** | 資源焦慮 | 金錢、投資、債務、薪資結構。 |
| **【情】** | 私密關係 | 戀愛、暗戀、婚姻、情感糾葛。 |
| **【業】** | 職涯發展 | 工作、創業、升遷、轉職、社會成就。 |
| **【家】** | 內部系統 | 夫妻、親子、婆媳、原生家庭、房產風水。 |
| **【人】** | 外部連結 | 朋友、職場小人、合夥、社交恐懼。 |
| **【身】** | 生理載體 | 健康、睡眠、外貌焦慮、體能狀態。 |
| **【學】** | 知識焦慮 | 考試、升學、證照、技能進修。 |
| **【運】** | 不確定性 | 流年、莫名不安、無法歸類之煩惱。 |

## 3. 第二階段（上）：內圈｜駕駛員掃描（Driver Status）

* **核心邏輯：** 測量「心理效能感 (E)」、「認知清晰度 (C)」與「行動潛力 (A)」。
* **運算規則：** 採用「三軸加權模型」。若選中【滯】，行動力強制扣減；若選中【虛】，權重高於意圖。

### 八字詞彙定義（內在狀態）

| 組別 | 詞彙 | 前台直覺 | 後台定義 (Backend) | 權重特徵 |
| --- | --- | --- | --- | --- |
| **能量** | **【 盛 】** | 氣勢強、有信心 | **High Agency** (高自我效能) | E軸高分，易衝動。 |
|  | **【 虛 】** | 好累、無力 | **Depletion** (自我耗損) | **(關鍵)** E軸負分，行動力歸零。 |
| **認知** | **【 覺 】** | 心裡清楚 | **Clarity** (高後設認知) | C軸高分，決策謹慎。 |
|  | **【 惑 】** | 資訊混亂 | **Ambiguity** (認知失調) | C軸負分，猶豫不決。 |
| **意圖** | **【 攻 】** | 想突破 | **Approach Motivation** | 僅代表意圖，受限於能量。 |
|  | **【 守 】** | 想維持 | **Avoidance Motivation** | 僅代表意圖，傾向保守。 |
| **流動** | **【 通 】** | 順暢 | **Flow** (心理流暢) | 全數值加成。 |
|  | **【 滯 】** | 卡卡的 | **Stagnation** (情緒阻滯) | **(關鍵)** 強制否決權，行動癱瘓。 |

> **輸出數據 (Internal Data)：**  
> `Driver_Score` = `{ Energy: ±N, Clarity: ±N, Action_Potential: ±N }`

## 4. 第二階段（下）：中圈｜路況掃描（Environment Hardness）

* **核心邏輯：** 測量外在環境的「硬度 (Hardness, H)」與「急迫度 (Urgency, U)」。
* **技術說明：** 採用「通用苦難原型」，依據第一階段的「戰場」自動生成具體解釋。
* **隱藏邏輯：** 底層保留精準的病理/物理定義，但前台經過修飾。

### 八字詞彙定義（外在困境）

| 詞彙 | 原型定義 | H (硬度) | U (急迫) | 底層邏輯範例 (身/財) |
| --- | --- | --- | --- | --- |
| **【 匱 】** | 稀缺/空無 | 5 | 2 | 負債過高 / 氣血兩虛、精氣耗竭。 |
| **【 阻 】** | 障礙/不通 | 4 | 1 | 資金凍結 / 血栓、結石、腫瘤。 |
| **【 爭 】** | 衝突/對立 | 3 | **5** | 官司訴訟 / 急性發炎、病毒感染。 |
| **【 亂 】** | 無序/混雜 | 2 | 3 | 帳目不清 / 內分泌失調、自律神經失調。 |
| **【 險 】** | 危機/動盪 | 4 | **5** | 瀕臨破產 / 心肌梗塞、急診重症。 |
| **【 孤 】** | 斷裂/無援 | 3 | 1 | 求貸無門 / 憂鬱傾向、神經連結斷裂。 |
| **【 遲 】** | 延宕/錯過 | 2 | 0 | 貨款被壓 / 代謝緩慢、老化。 |
| **【 剝 】** | 侵蝕/耗損 | 3 | 4 | 通膨侵蝕 / 慢性病拖延、癌症耗損。 |

> **輸出數據 (Internal Data)：**  
> `Env_Score` = `{ Hardness_Sum: 0~15, Urgency_Sum: 0~15, Risk_Level: Critical/Stable }`

## 5. 第三階段：外圈｜行為掃描（Executed Strategy）

* **核心邏輯：** 捕捉使用者「已付出資源」的實際行為（非意圖）。
* **演算法：** **GAP 策略適配度模型**。計算行為的「成本」是否大於「內在能量」。

### 八字詞彙定義（行為策略）

| 詞彙 | 行為定義 | 成本係數 () | 時間敏感度 () | 備註 |
| --- | --- | --- | --- | --- |
| **【 戰 】** | 已對抗、已投入資金 | **5 (極高)** | 快 | 高耗能，需高能量支撐。 |
| **【 避 】** | 已撤離、已拒絕 | 1 (低) | 快 | 止損策略。 |
| **【 忍 】** | 持續承受、不反應 | **4 (高內耗)** | 慢 | 極易導致資源透支。 |
| **【 謀 】** | 尚在規劃、分析 | 2 (中) | **極慢** | 對高急迫 (U) 環境有懲罰。 |
| **【 合 】** | 已找人幫忙 | 2 (中) | 中 | 引入外部資源。 |
| **【 斷 】** | 已切割、已離職 | 3 (中高) | **極快** | 激進止損。 |
| **【 隨 】** | 放任、隨波逐流 | 0 (無) | 慢 | 低控制感。 |
| **【 轉 】** | 已轉念、自我調適 | 1 (低) | 快 | 認知重構。 |

## 6. 第四階段：定錨決策與輸出（Synthesis & Output）

這是系統的最終結算層，負責將上述複雜運算轉化為使用者能聽懂的「建議」。

### 6.1 GAP 運算邏輯（判定吉凶）

系統將比對 `Action_Potential` (內在底氣) 與 `Cost_Factor` (行為成本)，並參考 `Hardness` (環境難度)。

* **協同 (Synergy)：** 能量足 + 策略對 + 路況通 → **建議：乘勝追擊。**
* **磨損 (Friction)：** 能量普 + 策略慢 + 路況亂 → **建議：調整節奏。**
* **崩塌 (Collapse Risk)：** 能量虛 + 策略戰 + 路況險 → **建議：緊急煞車（保命優先）。**

### 6.2 陰陽定錨（建議方向分類）

根據判定結果，系統將建議歸類為兩大方向，作為使用者下一步的行動指引：

**A. 偏陰（向內修 / Yin Direction）**

* **觸發條件：** 環境硬度 (H) 低，但內在能量 (E) 極低或認知 (C) 混亂。
* **診斷：** 「事情其實不難，是心態崩了。」
* **建議關鍵字：** 養（休養）、覺（釐清）、轉（轉念）。
* **具體解方：** 重建自信、調整動力、改變觀點、暫時抽離。

**B. 偏陽（向外衝 / Yang Direction）**

* **觸發條件：** 內在能量 (E) 高，但環境硬度 (H) 高或行為 (Strategy) 錯誤。
* **診斷：** 「心態很穩，是環境太難或方法不對。」
* **建議關鍵字：** 謀（佈局）、合（借力）、斷（停損）。
* **具體解方：** 尋找外部資源、更換戰術、等待時機。

## 7. 專案優勢與開發結論

1. **結構完整：** 從「鎖定問題」到「狀態檢測」再到「行為校正」，邏輯閉環完整。
2. **去醫療化合規設計：**
   * 後台運算使用精準醫學/心理學邏輯（如：Depression, Inflammation）。
   * 前台輸出轉譯為玄學/管理學語言（如：虛、滯、資源透支），符合平台規範且降低使用者防衛心。
3. **避免巴納姆效應（Barnum Effect）：** 系統不給模稜兩可的廢話，而是基於「內在 vs. 行為」的落差（GAP），給出具有優先級（Priority）的具體策略。
4. **工程可行性：** 所有的形容詞（如：很累、很急）都已轉化為可計算的權重數值（-5 ~ +5），開發團隊可直接依此撰寫演算法。
