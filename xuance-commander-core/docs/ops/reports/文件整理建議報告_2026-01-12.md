## 二、AI 工作習慣分析

### 2.1 AI 在沒有上下文時的需求

1. **快速找到關鍵資訊**：
   - 需要一個「總入口」文件，告訴 AI 應該先讀什麼
   - 需要清晰的優先級分類（哪些是必須讀的，哪些是可選的）

2. **理解當前狀態**：
   - 需要知道「現在在做什麼」
   - 需要知道「有哪些任務在進行中」
   - 需要知道「有哪些任務已完成」

3. **理解工作規則**：
   - 需要知道「應該遵守哪些制度」
   - 需要知道「文件應該放在哪裡」
   - 需要知道「任務應該怎麼執行」

4. **理解目標與進度**：
   - 需要知道「終極目標是什麼」
   - 需要知道「整體進度如何」
   - 需要知道「下一步應該做什麼」

### 2.2 目前架構的問題

1. **缺乏統一的「總入口」**：
   - 雖然有 `memory/index/COMMANDER_ENTRYPOINTS.md`，但它只指向 5 個文件
   - 沒有告訴 AI「如果完全沒有上下文，應該先讀什麼」

2. **文件散落**：
   - 進度報告、任務報告散落在根目錄和各任務資料夾
   - 時間戳記檔案會累積，造成混亂

3. **索引過於龐大**：
   - `GOVERNANCE_INDEX.md` 有 350+ 行，難以快速找到關鍵制度
   - 缺乏優先級分類

4. **缺乏「當前狀態」總覽**：
   - 雖然有 `CURRENT.md`，但它只包含短摘要
   - 缺乏完整的「當前任務狀態」總覽

**內容結構**：
```

## 第一次接觸這個專案時，請依序閱讀：

### 1. 終極目標（必須讀）
- charter/CHARTER.md（專案憲章）

### 2. 當前狀態（必須讀）
- memory/briefs/CURRENT.md（當前狀態摘要）
- docs/ops/TASK_STATUS.md（任務狀態總覽，見 3.2）

### 3. 工作規則（必須讀）
- docs/governance/ESSENTIAL_RULES.md（核心制度，見 3.3）
- docs/governance/CURSOR_FILE_PLACEMENT_RULE.md（文件放置規範）

### 4. 進度與目標（必須讀）
- roadmap/ROADMAP.md（主線進度）
- roadmap/MASTER_WORK_OBJECTIVES.md（主線工作目標）

### 5. 工作流程（建議讀）
```

**放置位置**：`docs/ONBOARDING.md`

### 3.2 統一進度與任務狀態追蹤

**建議**：建立 `docs/ops/TASK_STATUS.md`（任務狀態總覽）

**內容結構**：
```
# 任務狀態總覽（Task Status Overview）

## 當前進行中的任務
- [列出所有進行中的任務，包含狀態、進度、關鍵成果]

## 待執行的優先任務
- [列出所有待執行的優先任務，包含優先級、依賴關係]

## 已完成的任務
- [列出最近完成的任務，包含完成日期、關鍵成果]

## 詳細記錄
- [指向各任務的詳細記錄文件]
```

**更新機制**：
- 每次任務狀態變更時更新
- 定期（每週）整理，移除過期資訊
- 引用各任務資料夾中的詳細報告，不重複內容

**整合現有文件**：
- 整合 `MASTER_PROGRESS_REVIEW_2026-01-11.md` 的內容
- 整合 `目前任務狀態報告_2026-01-12.md` 的內容
- 整合 `PRIORITY_TASKS_LIST_2026-01-11.md` 的內容
- 整合 `docs/ops/TASK_RECORDS_INDEX.md` 的內容

**清理根目錄**：
- 將時間戳記檔案（如 `*_2026-01-11.md`）移動到 `docs/ops/archive/` 或刪除
- 只保留 `docs/ops/TASK_STATUS.md` 作為唯一入口

### 3.3 建立核心制度文件

**建議**：建立 `docs/governance/ESSENTIAL_RULES.md`（核心制度）

**目的**：讓 AI 快速了解必須遵守的核心制度，不需要讀完 70+ 個治理文件

**內容結構**：
```
# 核心制度（Essential Rules）

## 必須遵守的核心規則（按優先級排序）

### P0（最高優先級，每次工作前必須確認）
1. TEXT_ONLY_EXECUTION_RULES.md（文本主控硬規則）
2. CURSOR_FILE_PLACEMENT_RULE.md（文件放置規範）
3. PREWRITE_STATE_CONFIRMATION.md（寫入前現況確認規則）
4. PROGRESS_RECALIBRATION.md（進度重新校準規則）

### P1（高優先級，執行任務時必須遵守）
1. AI_ROLES_CURSOR_ARCHITECTURE.md（AI 角色架構）
2. CURSOR_TASK_PACKAGE_RULE.md（任務包規則）
3. TASK_COMPLETION_REPORT_PROTOCOL.md（任務完成報告協議）

### P2（中優先級，需要時查閱）
1. [其他重要制度...]

## 詳細制度索引
- 完整制度清單見：GOVERNANCE_INDEX.md
```

**更新機制**：
- 當有新的核心制度時，更新此文件
- 定期（每月）檢視，確保核心制度是最新的
