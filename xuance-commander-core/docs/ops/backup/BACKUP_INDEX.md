# 備份索引

**建立日期**：2026-01-12  
**用途**：記錄所有備份操作，用於恢復檔案

---

## 📋 備份規則

### 備份時機
- 在刪除任何檔案前，必須先建立備份
- 在修改重要檔案前，建議建立備份

### 備份位置
- 備份檔案存放在：`docs/ops/backup/YYYYMMDD_HHMM/`
- 備份索引：`docs/ops/backup/BACKUP_INDEX.md`

### 備份格式
- 保留原始檔案路徑結構
- 備份檔案命名：`原路徑/檔案名`
- 備份索引記錄：日期、時間、操作類型、原始路徑、備份路徑

---

## 📝 備份記錄

### 備份記錄格式
```markdown
## YYYY-MM-DD HH:MM - 操作類型

- **操作**：刪除/修改
- **原始路徑**：`path/to/file.md`
- **備份路徑**：`docs/ops/backup/YYYYMMDD_HHMM/path/to/file.md`
- **原因**：說明為什麼需要這個操作
- **恢復方式**：如何恢復這個檔案
```

---

## 🔄 恢復方式

### 使用恢復腳本
```bash
bash tools/restore_backup.sh <備份日期時間>
```

### 手動恢復
1. 找到對應的備份目錄：`docs/ops/backup/YYYYMMDD_HHMM/`
2. 找到需要恢復的檔案
3. 複製到原始位置
