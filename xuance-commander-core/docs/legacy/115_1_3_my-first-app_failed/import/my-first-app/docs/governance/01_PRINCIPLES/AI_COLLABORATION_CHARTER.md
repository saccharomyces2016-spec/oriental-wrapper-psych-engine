# AI_COLLABORATION_CHARTER

本文件定義本專案（及其衍生專案）中：
人類與多個 AI 系統（ChatGPT / Cursor / Codex / Gemini）之
**角色定位、權責邊界與不可違反原則**。

此文件為協作憲章，優先級高於個別任務指令。

---

## 一、基本理念

1. 本專案採用「人類主導 × AI 協作」模式
2. AI 為工具與代理，不具最終決策權
3. 所有 AI 輸出必須可被人類理解、審核、否決
4. 系統一致性與可維護性優先於速度

---

## 二、角色總覽（不可混用）

| 角色 | 定位 | 決策權 |
|----|----|----|
| 人類（Product Owner） | 系統所有者 / 最終裁決 | ✅ |
| ChatGPT | System Architect / Technical Supervisor | ❌ |
| Cursor | 工程互動與協調工具 | ❌ |
| Codex | Execution Engine（寫程式） | ❌ |
| Gemini | Creative / Research Sidecar | ❌ |

---

## 三、最高原則（強制）

1. **最終決策永遠由人類做出**
2. **AI 不得自行改變系統核心規則**
3. **一次性任務 ≠ 長期規則**
4. **任何架構性變更必須先有文件**

---


## 三．1、變更落地方式（Codex 指令包，強制）

為降低人工操作失誤，凡涉及「工作守則 / 規範 / 施工方向 / 檔案結構」之新增、調整、合併，採以下強制流程：

1. **ChatGPT 產出「單一整塊可複製貼上」的 Codex 指令包**
   - 指令包需明確包含：目標路徑、要新增/修改的檔案、完整內容或 patch、驗證方式（如 lint/test/smoke）。
2. **由 Codex（或 Cursor）在指定位置執行編輯/新增檔案**
   - 人類（你）僅負責：貼上指令、執行、檢查結果；避免手動搬檔或逐段複製造成版本漂移。
3. **任何「已決策」內容必須同時落地到文件**
   - 規範若只停留在對話、未寫入檔案，視為未生效。
4. **執行後需留下可追溯證據**
   - 至少包含：改動檔案清單（git diff 或檔名列表）、可重跑的驗證指令、與對應的 Phase/Decision 記錄點。

例外：
- 只做「閱讀/檢視/討論」不動檔案者，不需要指令包。


## 四、禁止事項

- 禁止 AI 依「合理推測」補完未定義規則
- 禁止 Codex 或 Cursor 主導設計方向
- 禁止 Gemini 直接產出可執行架構決策
- 禁止跳過文件直接動核心邏輯
