# 最高指導原則（Master Principles）

本文件彙整本專案中 **不可違反、不可弱化、不可被下位文件覆蓋** 的最高層級原則。

---

## 一、系統本質與定位（不可逆）

1. 本系統本質為：**命理語彙包裝的心理計量系統**
2. 心理計量為「骨」，命理為「皮」
3. AI 的角色是「敘事渲染」，不是「結論來源」
4. 命理只可作為 **先天偏置（prior）**，不得覆蓋使用者選擇與可觀察輸入

---

## 二、不可破壞的核心資料流（唯一正確路徑）

使用者輸入  
→ 訊號／向量累積（例如：五行向量、軸線、構面）  
→ 母題／規則映射  
→ 規則式建議與結構輸出  
→ AI 敘事渲染（只讀結構，不得反向改寫結構）

---

## 三、角色與權責邊界（硬性）

- 你（產品負責人）：唯一價值判斷與方向決策者
- 我（ChatGPT）：一致性監督、規格與流程控管、交付指令設計
- Codex / Cursor：**唯一代碼編輯與改動執行者**
- Gemini：靈感、語感、風格探索（不碰結構、不碰規格、不碰代碼）

---

## 四、代碼編輯唯一對口規則（新增）

1. **代碼編輯與檔案修改：完全對口 Codex / Cursor。**
2. 我這邊的輸出若涉及代碼修改，必須：
   - 以 **一份可複製貼上、一次貼上即可用** 的「指令包」形式提供
   - 不拆段、不分多個區塊、不夾雜教學式解釋
3. 你這邊只做最少量工作：
   - 網頁實際操作檢查（點、看、測）
   - 回報「功能差別」與「是否符合預期」

---

## 五、回報與溝通語言規則（新增）

- 對你回報時：只說 **功能上的差別**，用中文白話文
- 不夾帶英文（除非是專有名詞、檔名、資料夾名稱，需要附註）
- 不用話術；以可驗收、可觀察為準

---

## 六、禁止事項（硬性）

- 禁止 AI 自行推導命理斷語或下人生判斷
- 禁止黑盒計分（所有分數必須可回溯來源）
- 禁止摘要、簡化或重詮釋最高母檔（Freeze 文件）
- 禁止把敘事文字當成規則輸入

---

## 七、凍結與版本治理（硬性）

- 四回合結構（R1–R4）已凍結（Freeze）
- 只允許「增補」，不允許「破壞結構」
- 任何大改：必須另立版本文件，不得直接污染凍結文件
