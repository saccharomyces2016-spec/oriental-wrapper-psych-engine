# DESTINY_CODE（Canonical 重建資料夾）

本資料夾是「文件治理優先」的重建骨架：先把世界觀、規格、題庫工程與推導邏輯整理成可治理的文件系統，再逐步落地程式碼。

## 核心閱讀入口
- `docs/00_canonical/00_CONSTITUTION.md`：最高層原則與裁決規則（含協作裁決規則）
- `docs/00_canonical/01_PROJECT_MASTER.md`：專案總規格與重建藍圖
- `docs/00_canonical/02_QUESTION_SYSTEM.md`：四回合題庫與計分推導規格
- `docs/00_canonical/03_THEORY_FOUNDATION.md`：心理結構母檔、出題工程、五行資料庫（隱喻/映射）
- `docs/00_canonical/04_GOVERNANCE_EVIDENCE.md`：治理狀態與事實抽取（證據層）

## 原始檔保存
- `docs/99_archive_origin/`：你上傳的所有原始檔案（原封不動備份，便於回溯與比對）
  - 注意：此資料夾內可包含多版本或時間戳版本；是否刪除舊版需經決策後才做。

## 程式碼骨架（尚未動工）
- `packages/core/`：未來放核心引擎與題庫資料（JSON/TS/JS）
- `packages/web/`：未來放互動式網頁前端（UI/流程）
- 目前僅建立空目錄，方便後續與 Codex/Cursor 對接落地。

## 決策紀錄
- `docs/01_decisions/`：每次針對衝突條款/版本裁決，新增一份決策紀錄（不要覆寫舊紀錄）。
