## 五、【顧問師任務包 1】— R2 敘事設計顧問

### 任務包：P0-4 / R2

#### 任務目標
為 `relationship_imbalance` Facet 定義**唯一主隱喻**與**次隱喻白名單**，以驗證 P0-3 隱喻治理機制可零改動搬移。

#### 任務內容（必須逐條符合）

**1. 定義主隱喻 1 個**
- 能表達「關係長期失衡的狀態感」
- 屬東方玄學／自然秩序語系
- 不得與 P0-3 主隱喻重疊（P0-3 使用「歲時農耕・倉廩觀」）
- 不得使用心理、關係、法律、管理語言
- 符合文明一致性原則（參考 `docs/domain/advisory/GOVERNANCE_METAPHOR_COMPATIBILITY_RULE.md`）

**2. 建立次隱喻白名單**
- 可被 grep / regex 機器檢查
- 每段最多 1 個次隱喻
- 禁止角色或場景替換
- 明確列出：允許 / 禁止 類型
- Facet 限定，不構成全域禁用

**3. 建立使用規則**
- Facet 限定
- 不構成全域禁用
- 與 P0-3 規則等價，不新增原則

#### 禁區（命中即 FAIL）
- ❌ 解釋原理
- ❌ 提供建議
- ❌ 心理學、溝通學、關係術語
- ❌ 任何形式的結論或判決

#### 參考範例
- `docs/domain/advisory/R2/P0-3_SECONDARY_METAPHOR_WHITELIST.md`（P0-3 的次隱喻白名單範例）
- `docs/domain/advisory/R2/P0-3_SECONDARY_METAPHOR_RULES.md`（P0-3 的機器化檢查規則範例）

#### 交付物（寫入路徑）
- `docs/domain/advisory/R2/P0-4_SECONDARY_METAPHOR_WHITELIST.md`（新建）
- `docs/domain/advisory/R2/P0-4_SECONDARY_METAPHOR_RULES.md`（新建）

#### 驗收標準
- ✅ 能直接套入 P0-3 檢查器
- ✅ 不需修改任何既有規則
- ✅ 換 Facet 但結構完全一致

## 六、【顧問師任務包 2】— R4 風險鏈顧問

### 任務包：P0-4 / R4

#### 任務目標
為 `relationship_imbalance` Facet 補齊**安全邊界**：禁用詞擴充 + L4 高風險出口模板，確保高壓情境不失控。

#### 任務內容

**1️⃣ 禁用詞擴充**
- 擴充 Facet 專用禁用詞（CN/EN）
- 類型至少包含：
  - 心理 / 醫療
  - 法律 / 關係判決
  - 現實決策指令
  - 宿命 / 恐嚇
- 規則：命中即 FAIL，不轉譯、不補救
- 可被 grep / regex 機器檢查

**2️⃣ L4 高風險出口模板（1–2 條）**
- 僅在高風險觸發時使用
- 語氣要求：
  - 不恐嚇
  - 不宿命
  - 不下具體決策（分手、離婚等）
  - 僅描述「態勢 / 走向 / 宜忌」
  - 必須可收束，不引導延伸
- 允許少量變數插槽（但必須符合主隱喻）

#### 禁區（命中即 FAIL）
- ❌ 推斷對方行為或動機
- ❌ 現實決策指令
- ❌ 心理、法律、醫療語言

#### 參考範例
- `docs/domain/advisory/R4/P0-3_NARRATIVE_BLACKLIST_SSOT.json`（P0-3 的禁用詞表範例）
- `docs/domain/advisory/R4/P0-3_BANNED_TERMS_LIST.md`（P0-3 的禁用詞表人類可讀版本範例）
- `docs/domain/advisory/R4/P0-3_L4_HIGH_RISK_EXIT_TEMPLATES.md`（P0-3 的 L4 高風險出口模板範例）

#### 交付物（寫入路徑）
- `docs/domain/advisory/R4/P0-4_NARRATIVE_BLACKLIST_SSOT.json`（新建或更新）
- `docs/domain/advisory/R4/P0-4_BANNED_TERMS_LIST.md`（新建）
- `docs/domain/advisory/R4/P0-4_L4_HIGH_RISK_EXIT_TEMPLATES.md`（新建）

#### 驗收標準
- ✅ 高風險只能走模板
- ✅ 模板語氣穩定、可收束
- ✅ 與 P0-3 安全哲學完全一致
