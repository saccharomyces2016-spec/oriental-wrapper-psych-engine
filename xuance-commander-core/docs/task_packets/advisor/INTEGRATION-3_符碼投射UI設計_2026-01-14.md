## 📚 核心概念

### 符碼投射（Symbolic Projection）

**定義**：透過符號、情境、隱喻讓使用者投射自己的狀態，而不是直接問「你焦慮嗎？」

**範例對比**：

**傳統心理學方式**：
```
Q: 你最近是否經常感到焦慮？
A: [ ] 從不  [ ] 偶爾  [ ] 經常  [ ] 總是
```

**符碼投射方式**：
```
【情境】：你站在十字路口，前方有四條路。

🌊 左邊的路霧氣瀰漫，看不清前方
🔥 右邊的路火光沖天，熱氣逼人
🌳 前方的路草木茂盛，但荊棘叢生
⛰️ 後方的路一片平坦，但通往深山

請選擇你會走的路：________

→ 底層對應：
  🌊 = Water_Excessive（情緒滲透、不確定感）
  🔥 = Fire_Excessive（能量外耗、焦躁）
  🌳 = Wood_Excessive（過度行動、衝勁）
  ⛰️ = Earth_Excessive（迴避應對、停滯）
```

### 檔案 1：`SYMBOLIC_PROJECTION_DESIGN_PRINCIPLES.md`

**內容**：符碼投射設計原則

```markdown
# 符碼投射設計原則

## 1. 核心原則

### 原則 1：隱喻優於直述
- ❌ 不要問：「你是否經常感到焦慮？」
- ✅ 而要問：「你的內在是否像一團紊亂的風？」

### 原則 2：情境優於選項
- ❌ 不要用：A/B/C/D 四選一
- ✅ 而要用：情境描述 + 符號選擇

### 原則 3：象徵優於文字
- ❌ 不要用：「選擇 1-5 分」
- ✅ 而要用：🌊🔥🌳⛰️💎 符號

### 原則 4：互動優於靜態
- ❌ 不要用：點擊按鈕
- ✅ 而要用：拖曳、旋轉、觸碰

## 2. 五行符號系統

| 五行 | 符號 | 顏色 | 動畫效果 |
|------|------|------|---------|
| 水 | 🌊 | 藍色 | 流動、波紋 |
| 火 | 🔥 | 紅色 | 燃燒、跳動 |
| 木 | 🌳 | 綠色 | 生長、搖曳 |
| 金 | 💎 | 白色/金色 | 閃爍、堅硬 |
| 土 | ⛰️ | 黃色/棕色 | 穩定、沉重 |

## 3. 八卦符號系統

| 八卦 | 符號 | 視覺呈現 |
|------|------|---------|
| 乾 | ☰ | 三實線（天） |
| 坤 | ☷ | 三虛線（地） |
| 坎 | ☵ | 中實線（水） |
| 離 | ☲ | 中虛線（火） |
| 震 | ☳ | 下實線（雷） |
| 巽 | ☴ | 下虛線（風） |
| 艮 | ☶ | 上實線（山） |
| 兌 | ☱ | 上虛線（澤） |

## 4. 玄學感營造方法

### 視覺風格
- 色彩：黑、白、金、紅、藍（傳統玄學配色）
- 字體：書法體、宋體（避免無襯線體）
- 紋理：水墨、雲霧、山水

### 動畫效果
- 水流：波紋、流動、漣漪
- 火焰：燃燒、跳動、明滅
- 雲霧：飄散、聚攏、變幻
- 符號：旋轉、浮現、消隱

### 音效
- 背景音：古琴、流水、風聲
- 互動音：鐘聲、木魚、銅鈴
- 結果音：鼓聲、磬聲
```

```markdown
# 起卦互動設計

## 1. 傳統起卦方式

### 擲筊法
- 三枚銅錢
- 擲 6 次
- 記錄陰陽

### 數位化起卦
- 時間起卦
- 姓名起卦
- 隨機數起卦

## 2. 我們的起卦設計

### 方案 1：虛擬擲筊
**UI 設計**：
- 畫面中央：三枚銅錢（3D 模型）
- 操作：點擊「擲筊」按鈕或搖動手機
- 動畫：銅錢旋轉、落地、靜止
- 結果：顯示陰陽，記錄爻象

**對應邏輯**：
- 隨機生成 → 對應到八卦領域
- 例如：☵☳（坎上震下）→ 對應到「流動資源 + 行動決策」的綜合領域

### 方案 2：符號選擇
**UI 設計**：
- 畫面：8 個八卦符號環繞成圓
- 操作：選擇最吸引你的八卦
- 動畫：選中的八卦放大、發光

**對應邏輯**：
- 直接對應到八卦領域
- 例如：選擇 ☵（坎卦）→ 進入「流動資源」領域

### 方案 3：情境投射
**UI 設計**：
- 畫面：一幅山水畫
- 畫中有 8 個位置（山頂、水邊、森林等）
- 操作：選擇你最想去的位置

**對應邏輯**：
- 山頂 → 乾（主導創始）
- 水邊 → 坎（流動資源）
- 森林 → 巽（關係溝通）
- 等等

### 建議方案
推薦**方案 2（符號選擇）+ 方案 3（情境投射）組合**：
1. 先呈現情境投射（降低玄學門檻）
2. 然後顯示對應的八卦符號（增強玄學感）
```
