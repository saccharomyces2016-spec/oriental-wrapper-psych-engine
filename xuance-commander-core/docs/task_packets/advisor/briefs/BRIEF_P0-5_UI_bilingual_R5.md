### UI 架構設計（參考）

#### UI 定位
- **UI = 敘事管線（Narrative Pipeline）**
- **UI = 儀式引導器（Ritual Orchestrator）**
- 不是 App、不是問卷、不是心理測驗
- 是一個：「進入玄學世界 → 自我投射 → 被系統照見 → 得到指引」的儀式性體驗

#### UI 分層架構
- **Layer 0**：世界觀與氣氛層（開場動畫、世界觀短敘事、語言切換）
- **Layer 1**：使用者錨定層（姓名、性別、出生年月日）
- **Layer 2**：困擾領域選擇層（Facet Gate，卡片式困境選擇）
- **Layer 3**：問題行走層（Question Walk，逐題顯示）
- **Layer 4**：轉譯儀式層（Transition Ritual，運算中的象徵階段）
- **Layer 5**：結果敘事層（Narrative Stage，主敘事、行動建議、風險鏈）
- **Layer 6**：指引與收尾層（Guidance & Exit，不做決策，只留下方向）

**詳細規格**：見 `P0-5/P0-5_UI_ARCHITECTURE.md`

## 常見問題（FAQ）

### Q1: 如果 R2 尚未產出，我應該做什麼？
**A**: 可以先準備關鍵敘事詞彙對照表和轉譯策略，這些是通用的，不依賴 R2 的具體產出。

### Q2: 如何判斷英文版本是否「過於像心理測驗」？
**A**: 如果英文版本使用了過多的「self-help」、「coaching」、「psychology」語彙，或者過於「分析性」，就可能被視為心理測驗。應該使用更「神秘」、「儀式性」的語彙。

### Q3: 如何確保中英文敘事風格一致但不僵硬？
**A**: 重點是保持「神秘感」和「可信度」的一致性，而不是逐字翻譯。可以使用不同的語彙和表達方式，但必須在文化感知層保持一致。
