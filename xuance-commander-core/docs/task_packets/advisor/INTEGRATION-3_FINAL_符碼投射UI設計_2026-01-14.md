## 📂 交付物詳細說明（10 檔案摘要）

### 1. 設計原則與規範

**檔案 1：`SYMBOLIC_PROJECTION_DESIGN_PRINCIPLES.md`**

**儀式感重塑**：所有 UI 動效必須遵循「慢、靜、不可控」原則。去除所有「科技回饋感」（如按鈕點擊聲、高頻閃爍）。

**美學定義**：採用「數位古物（Digital Antiquity）」風格——材質需呈現時間的磨損感，光影需呈現燭火般的呼吸感。

---

### 2. 核心互動流程

**檔案 2：`QIGUA_INTERACTION_DESIGN.md` (Stage 1: 叩門)**

**設計**：八扇懸浮於虛空中的「心門」

**互動**：使用者**長按太極**注入意念，隨後**靜置等待**。八門隨時間流轉，門序自成，不可預期。最終某一扇門縫自然顯現光影與判詞（如「陷落」），示意使用者進入。

**產出**：鎖定 `Domain` (八卦領域)

---

**檔案 3：`LIUQIN_SELECTION_DESIGN.md` (Context Injection)**

**設計**：將六親轉化為環境意象

**互動**：環境中自然浮現的物象（如：層層疊疊的屋簷代表父母爻的規範）。使用者不是去「選」，而是去「注視」那個讓他感到壓力的景象。

**產出**：鎖定 `Context` (壓力來源)

---

**檔案 4：`PROJECTION_QUESTION_LIBRARY.md` (Stage 2: 辭象)**

**重大變革**：將「點選詞彙」改為「辭象感應」

**互動**：詞彙如煙霧般在虛空中緩慢生滅。使用者被引導**「於緩慢浮現的辭象中，任由那些觸動心弦者停留於眼前」**。辭象停留，即為心念所繫。

**運算**：後端將詞彙對應的 `Vector` 進行隱性疊加

---

**檔案 5：`FUMA_PROJECTION_DESIGN.md` (Time/Space)**

**設計**：12 地支羅盤，用於捕捉「時空/應期」參數

**互動**：羅盤緩慢自轉，使用者在心中默念問題，憑直覺令其**「定止」**。文字浮現（如 子水 -> 深夜/潛藏），作為輔助校正參數。

## ⚙️ 技術對接驗證

### Q: 如何在「去賽博化」的同時，確保精準度？

我們採用 **「隱性向量疊加」** 技術，將運算完全隱藏在儀式之後：

**象徵層 (User View)**：
- 使用者看到的是：「溺斃」、「冰封」、「漩渦」等辭象在眼前停留
- 體驗：這是在描述我的夢境或感受

**邏輯層 (Hidden Engine)**：
- 資料庫映射：`"溺斃" -> { Facet: "Kan_Depletion", Severity: 1.5, Rigidity: "Locked" }`
- 運算：系統靜默進行向量相加
- 結果：系統轉譯為「其勢已近臨界，需引導而非強斷」

**結論**：使用者不需要知道自己得了幾分，他只需要知道「卦象懂我」。
