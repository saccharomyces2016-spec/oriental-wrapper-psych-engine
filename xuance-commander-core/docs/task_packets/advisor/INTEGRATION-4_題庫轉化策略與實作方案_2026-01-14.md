## 🎯 核心問題

### 關鍵決策：如何從底層題庫轉化到符碼投射UI？

**方案 A：另建表層題庫**
- 創建全新的符碼投射題庫
- 與底層 OCTET_8 獨立維護
- 需要建立對應關係表

**方案 B：提煉現有題庫**
- 直接從現有 184 題中提煉
- 轉化為符碼投射形式
- 維護成本低

---

## 📊 策略分析與建議

### 建議方案：**混合策略（70% 提煉 + 30% 新建）**

**理由**：
1. ✅ **現有題庫品質優秀**，不應浪費
2. ✅ **符碼投射需要新的表達形式**，但核心邏輯相同
3. ✅ **混合策略平衡效率與品質**

## 🔄 轉化策略詳解

### 策略 1：直接提煉（70% 題目適用）

**原理**：將直述式問題轉化為辭象

**範例**：

**原題（OCTET_8）**：
```
Q1: 當你結束一天的工作或活動後，你的狀態是？
0 = 感到充滿活力，還想做點什麼
1 = 有點累，但休息一下就好
2 = 很累，需要躺下來休息
3 = 筋疲力盡，什麼都不想做
4 = 完全癱軟，連動都不想動
```

**轉化後（辭象投射）**：
```
【情境】：日落時分，你站在歸途上。
請選擇最接近你狀態的辭象：

💧 「餘力猶存」（0分）
🌙 「微倦待息」（1分）
🍂 「氣耗需養」（2分）
🕯️ 「燈油將盡」（3分）
⚫ 「能量枯竭」（4分）
```

**底層對應**：
- 使用者選擇「燈油將盡」→ 系統記錄 Q1 = 3 分
- 完全相同的診斷邏輯，不同的表達形式

---

### 策略 2：符號重構（30% 題目需新建）

**原理**：某些題目過於直述，需要重新設計為符號投射

**範例**：

**原題（OCTET_8）**：
```
Q7: 當你意識到自己這麼累時，你能否主動做些什麼改變？
0 = 可以，我會主動調整
1 = 可以，但需要一些時間
2 = 很難，不知道該怎麼做
3 = 幾乎無法，覺得無力改變
4 = 完全無法，只能這樣下去
```

**轉化後（符號投射）**：
```
【情境】：你站在岔路口，前方有三條路。

🌳 一條路有明顯的指示牌（可以主動調整）
🌫️ 一條路霧氣瀰漫，需要摸索前進（很難，不知道怎麼做）
⛰️ 一條路被巨石擋住（完全無法改變）

請選擇你會走的路：________
```

**底層對應**：
- 🌳 = 0-1分（高能動性）
- 🌫️ = 2分（中能動性）
- ⛰️ = 3-4分（低能動性）

# 題目轉化策略

## 1. 轉化原則

### 原則 1：保持診斷邏輯不變
- ✅ 底層計算完全相同（OCTET_8, Severity, Rigidity）

- ❌ 不用：「你是否經常...」（直述式）

- 使用八卦符號（☰☷☵☲☳☴☶☱）
- 使用天干地支符號

## 3. 品質控制

### 必須保證
- ✅ 每個辭象的分數對應唯一
- ✅ 使用者選擇後，系統能精準記錄 0-4 分
- ✅ 轉化後的題目仍然具有區辨力

## 3. 神秘感營造（類抽卡體驗）

### 核心機制：「等待顯化」

**流程**：
```
1. 使用者長按太極（1-2 秒）
   ↓
2. 辭象逐一浮現（每個間隔 0.5 秒）
   ↓
3. 使用者凝視，選擇觸動心弦的辭象
   ↓
4. 選中效果：辭象放大 + 古銅色光暈
   ↓
5. 其他辭象淡出（opacity 0.3）
   ↓
6. 確認按鈕浮現：「以此為記」
```

**神秘感來源**：
- ✅ 等待時間（1-2 秒）→ 期待感
- ✅ 逐一浮現（不是一次全顯示）→ 揭示感
- ✅ 古典用詞（「以此為記」）→ 儀式感
- ✅ 視覺回饋（光暈）→ 被選中的感覺
